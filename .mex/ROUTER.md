- [`overlay-slot-bounded-config-save.md`](patterns/overlay-slot-bounded-config-save.md): SPCTLK4 config writers must bound every append before it reaches past the 512-byte `overlay_slot`; a final size check is not enough once memory has already been overwritten.
- [`overlay-resident-string-dedup.md`](patterns/overlay-resident-string-dedup.md): overlays should reuse resident exported string constants when the text is identical, to recover overlay headroom without changing UI or growing TAP.
- [`overlay-cold-status-text-shrink.md`](patterns/overlay-cold-status-text-shrink.md): cold overlay status text can be shortened when the semantic state is still clear and the freed bytes buy safety margin in a nearly full overlay.
- [`z80-ldai-iff-save.md`](patterns/z80-ldai-iff-save.md): avoid `ld a,i` IFF save/restore in routines that can tolerate a fixed interrupt state, and document the callsite contract when doing so.
- [`overlay-local-helper-relocation.md`](patterns/overlay-local-helper-relocation.md): if a helper is only imported by one overlay, first free overlay headroom, then move the helper into that overlay and remove it from the resident ABI.
- [`fast-text-renderer-alignment.md`](patterns/fast-text-renderer-alignment.md): any bulk 64-col row renderer must reproduce the same scanline padding as the per-character renderer, or redrawn text will jump vertically relative to incrementally printed text and the cursor/prompt.
- [`frameless-asm-helper-reuse.md`](patterns/frameless-asm-helper-reuse.md): small resident frameless wrappers should call an existing helper instead of inlining the same byte loop again when the helper's callee stack contract can sit above the wrapper's deeper saved context without disturbing it.
- [`resident-asm-helper-reuse.md`](patterns/resident-asm-helper-reuse.md): in resident hand-written ASM, prefer shared IX load helpers and preserved register reuse over open-coded re-materialization when the callee contract already proves the values are still live.
- [`rendering-hotpath-byte-rules.md`](patterns/rendering-hotpath-byte-rules.md): for rendering hot paths, exploit proven alignment/caching contracts, collapse byte-packed coordinate adds into 16-bit adds, and do not pay `IY` save/restore or cache revalidation that the callee contract already makes unnecessary.
- [`external-review-context-package.md`](patterns/external-review-context-package.md): share packages for external review should include a reusable prompt, preserve relative paths, and stay scoped to current active source plus relevant `.mex` constraints.
- [`demoscene-render-scroll-review.md`](patterns/demoscene-render-scroll-review.md): demoscene-inspired render/scroll ideas should be measurement-first; the current accepted low-risk changes are `_print_line64_fast()` blank-pair skipping and direct row-19 scroll clear, while stack/SMC/HALT/smooth-scroll remain prototype-only.
- [`bpe-aware-main-print-fastpath.md`](patterns/bpe-aware-main-print-fastpath.md): `main_print()` short-line fast paths must preserve the BPE-aware slow path by treating any byte `>=0x80` as a forced `_main_puts()` case.
- [`aggressive-pre-render-shrink.md`](patterns/aggressive-pre-render-shrink.md): before spending resident bytes on render/scroll experiments, recover slack through measured dead-code, ABI-narrowing, fixed-scratch, and small ASM rewrites, keeping rejected byte economics visible.
- [`sntp-retry-on-connect.md`](patterns/sntp-retry-on-connect.md): the `/server` connect path must retry `AT+CIPSNTPTIME?` up to 3 times with a delay, because the ESP8266 can return a `1970` placeholder before NTP has actually resolved; retries must use `sntp_queried` as the exit signal, stay inside the AT-command window, and honor `BREAK` across all attempts.
- [`keyboard-translation-table-shrink.md`](patterns/keyboard-translation-table-shrink.md): `_in_inkey()` should keep the explicit CAPS table unless a measured replacement beats it; deriving uppercase plus a small exception table currently grows the resident binary.
- [`input-cache-vram-attr.md`](patterns/input-cache-vram-attr.md): input redraw caching should cache characters only and validate attributes against real VRAM; the old fixed-RAM attribute cache was write-only.
- [`timestamp-indent-fastpath.md`](patterns/timestamp-indent-fastpath.md): the main timestamp/wrap indent is currently a fixed 6-column contract; shared direct clear helpers are valid only while every writer keeps `wrap_indent` to 0 or 6.
- [`scroll-border-profiling.md`](patterns/scroll-border-profiling.md): scroll timing should be measured with the optional `SCROLL_PROFILE=1` border build, not by leaving debug border writes in the release path.
